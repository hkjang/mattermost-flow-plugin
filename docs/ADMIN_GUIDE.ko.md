# 관리자 가이드

이 문서는 Flow 플러그인을 설치, 활성화, 운영하는 Mattermost 관리자와 보드 소유자를 위한 가이드입니다.

## 요구 사항

- 플러그인 매니페스트 최소 버전 기준 Mattermost 서버 `6.2.1` 이상
- Mattermost 서버에서 플러그인 업로드 활성화
- 플러그인 설치 및 활성화 권한

## 플러그인 설치

### 방법 1: 릴리즈 번들 업로드

1. [GitHub Releases](https://github.com/hkjang/mattermost-flow-plugin/releases)에서 `com.mattermost.flow-plugin-<version>.tar.gz`를 다운로드합니다.
2. `System Console -> Plugin Management`를 엽니다.
3. `.tar.gz` 번들을 업로드합니다.
4. 업로드 후 플러그인을 활성화합니다.

### 방법 2: `mmctl`로 설치

```bash
mmctl plugin add dist/com.mattermost.flow-plugin-<version>.tar.gz --local
mmctl plugin enable com.mattermost.flow-plugin
```

## 플러그인 업로드 활성화

플러그인 업로드가 비활성화되어 있으면 Mattermost 설정에서 켭니다.

```json
{
  "PluginSettings": {
    "EnableUploads": true
  }
}
```

환경에 따라 설정 적용 후 Mattermost 재시작이 필요할 수 있습니다.

## 권한 모델

Flow는 보드 접근 제어에 Mattermost 멤버십과 관리자 권한을 그대로 사용합니다.

- 보드 조회 사용자는 해당 팀 또는 채널 범위의 멤버여야 합니다.
- 보드 관리자는 보드 설정과 보드 구조를 수정할 수 있습니다.
- 팀 관리자는 자신의 팀 안의 보드를 관리할 수 있습니다.
- 시스템 관리자는 모든 보드를 관리할 수 있습니다.

보드 단위 관리자는 보드 메타데이터에 함께 저장됩니다.

## 운영 방식

### 저장소

Flow는 운영 데이터를 Mattermost 플러그인 KV 저장소에 저장합니다.

- 보드 메타데이터와 컬럼
- 카드와 의존성
- 활동 이력
- 사용자 보기 설정
- 채널 기본 보드 매핑
- 마감 임박 알림 상태
- 캘린더 피드 토큰

현재 구조에서는 외부 데이터베이스가 필요하지 않습니다.

### 알림

보드 설정에서 허용하면 Flow가 연결된 채널에 업데이트를 포스트할 수 있습니다.

보드 단위 주요 설정:

- `post_updates`
- `post_due_soon`
- `allow_mentions`
- `calendar_feed_enabled`
- `default_view`

마감 임박 스캔은 클러스터 백그라운드 작업으로 매시간 실행됩니다.

### 보드 진단

보드 소유자는 `Board settings` 안의 진단 패널에서 다음 항목을 점검할 수 있습니다.

- 카드가 존재하지 않는 컬럼을 참조하는지
- 카드 날짜 범위가 잘못되었는지
- 한 컬럼 안에서 수동 정렬 값이 중복되는지
- 의존성이 없는 카드로 연결되는지
- 자기 자신을 가리키는 의존성이 있는지

리포트에 `Reindex cards`가 활성화되면 안전한 복구 액션으로 카드 순서를 다시 정리하고, 고아 카드가 있으면 첫 번째 유효 컬럼으로 이동시킬 수 있습니다.

### 릴리즈 번들의 실행 권한

릴리즈 아카이브는 `server/dist/` 아래 파일이 실행 권한 `0755`로 저장되도록 패키징됩니다. 덕분에 Mattermost가 플러그인 번들을 풀었을 때 Linux나 macOS 서버에서 실행 파일 권한 문제로 서버 바이너리가 실행되지 않는 상황을 줄일 수 있습니다.

## 권장 롤아웃 순서

1. 스테이징 Mattermost 인스턴스에 플러그인을 설치합니다.
2. 팀 보드 하나와 채널 보드 하나를 만듭니다.
3. 보드 뷰, 대시보드 뷰, 간트 뷰, 슬래시 명령, 채널 포스트를 확인합니다.
4. 보드 설정의 진단 패널이 정상인지 확인합니다.
5. 외부 캘린더를 쓴다면 `.ics` 다운로드와 구독 URL도 검증합니다.
6. 운영 환경에서 플러그인을 활성화합니다.
7. 사용자 가이드와 보드 운영 규칙을 팀 소유자에게 공유합니다.

## 업그레이드와 롤백

- 업그레이드: 새 `.tar.gz` 번들을 업로드하거나 `mmctl`로 새 릴리즈를 설치합니다.
- 롤백: 이전 릴리즈 번들을 다시 업로드하고 필요하면 재활성화합니다.

Flow는 KV 저장소에 데이터를 보관하므로 롤백 시 플러그인 버전 차이가 너무 크지 않도록 유지하는 편이 안전합니다.

## 문제 해결

### 플러그인 업로드 실패

- `PluginSettings.EnableUploads`가 활성화되어 있는지 확인합니다.
- 업로드한 파일이 저장소 소스 압축본이 아니라 생성된 `.tar.gz` 번들인지 확인합니다.

### 사용자가 `Not authorized`를 받는 경우

- 사용자가 Mattermost에 로그인한 상태인지 확인합니다.
- 사용자가 해당 보드의 팀 또는 채널 멤버인지 확인합니다.
- 리버스 프록시가 플러그인 라우트의 Mattermost 인증 헤더를 제거하지 않는지 확인합니다.

### 마감 임박 포스트가 올라오지 않는 경우

- 보드가 채널 범위인지 확인합니다.
- 보드 설정에서 `post_due_soon`이 활성화되어 있는지 확인합니다.
- 카드에 마감일이 있고 이미 완료 상태가 아닌지 확인합니다.

### 빠른 액션에서 담당자 멘션이 되지 않는 경우

- 보드 설정에서 `allow_mentions`가 활성화되어 있는지 확인합니다.

### 진단에 고아 카드나 중복 순서가 보이는 경우

- 진단 패널에서 `Reindex cards`를 실행합니다.
- 최근 컬럼 변경이나 수동 데이터 수정이 있었는지 확인합니다.

### 캘린더 구독 링크가 동작하지 않는 경우

- 보드 설정에서 `calendar_feed_enabled`가 활성화되어 있는지 확인합니다.
- 이전 공유 링크를 끊어야 하면 보드 설정에서 토큰을 재발급하고 새 링크를 다시 배포합니다.
